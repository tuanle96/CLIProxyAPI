🔴 Root Cause: Compact 404 với Model DeepSeek

Lỗi: Error running remote compact task: unexpected status 404 Not Found

URL: https://cook.tuanle.dev/v1/responses/compact

📋 Tóm tắt

Model deepseek-v4-pro sử dụng OpenAI-Compatible executor, không phải Codex executor. Do đó, upstream fallback sang Codex KHÔNG được kích hoạt khi gọi /responses/compact.

🔍 Root Cause Chi Tiết

1️⃣ DeepSeek sử dụng OpenAI-Compatible Executor

File: openai_compat_executor.go:104-107

if opts.Alt == "responses/compact" {
    to = sdktranslator.FromString("openai-response")
    endpoint = "/responses/compact"
}

OpenAI-Compatible executor gửi trực tiếp request tới baseURL + "/responses/compact" của provider (DeepSeek).

2️⃣ DeepSeek Provider KHÔNG hỗ trợ /responses/compact

DeepSeek API chỉ hỗ trợ OpenAI-compatible endpoints:

  • /chat/completions ✅
  • /responses/compact ❌ (404 Not Found)

3️⃣ Codex Fallback CHỈ hoạt động với Codex Executor

File: codex_websockets_executor.go:177-179

if opts.Alt == "responses/compact" {
    return e.CodexExecutor.executeCompact(ctx, auth, req, opts)
}

Fallback này CHỈ tồn tại trong CodexWebsocketsExecutor, không có trong OpenAICompatExecutor.

💡 Giải pháp

Thêm Codex Fallback vào OpenAICompatExecutor

Khi OpenAI-compatible provider trả về 404 cho /responses/compact, tự động fallback sang Codex executor.

openai_compat_executor.go

func (e *OpenAICompatExecutor) Execute(...) {
    // ... existing code ...

    if httpResp.StatusCode == 404 && opts.Alt == "responses/compact" {
        // Fallback to Codex executor
        codexExec := NewCodexExecutor(e.cfg)
        return codexExec.executeCompact(ctx, auth, req, opts)
    }
}

1. Request: deepseek-v4-pro + compact

✅ Mong muốn (Fix)
